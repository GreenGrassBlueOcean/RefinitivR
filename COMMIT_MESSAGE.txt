feat: add progress reporting for long-running chunked requests

Replace the unusable verbose mode (full coordinator dump per chunk) with
a three-mode progress system controlled by `refinitiv_progress` option:

- TRUE (default): compact per-chunk progress for multi-chunk requests,
  silent for single-chunk ("one-chunk rule")
- FALSE: silent
- "verbose": compact progress + full coordinator dump (old behavior)

Output uses base R message() — portable to knitr/CI/Docker, suppressible
with suppressMessages(). Zero new dependencies.

Changes:
- Add progress_msg() helper and rewrite chunked_download() with
  caller_label, chunk_sizes, per-chunk timing, and completion summary
- Update EikonChunker() to use progress_msg() with richer format
  (instrument count, data points, chunk count)
- Standardize send_json_request() polling messages with [RefinitivR]
  prefix and attempt counts (force = TRUE so always visible)
- Add pagination progress to rd_GetOwnership()

  TRUE) in EikonGetData, EikonGetTimeseries, EikonGetSymbology,

- Add 26 tests covering all three modes, one-chunk rule, retry
